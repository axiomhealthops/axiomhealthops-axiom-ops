# Migrations

`20260528120000_baseline_2026_05_28.sql` is a one-time CAPTURE of the
production schema as it existed on 2026-05-28. It was generated by
querying the live DB's catalog tables (pg_class, pg_proc, pg_trigger,
pg_get_viewdef, etc.) and concatenating the result.

DO NOT re-run the baseline against an existing DB — it's intended for
recreating the schema from scratch (e.g., on a fresh branch DB or local
dev environment).

All subsequent migrations should be INCREMENTAL CHANGES — files like
`20260601_add_X_column.sql` that add/alter/drop on top of the baseline.
